`debian/` metadata, `branding.desc`, `settings.conf`, and `slideshow.qml` — but
all the actual PNG/theme files are missing (replaced by ASSETS.md placeholders).

## Script design (`generate-assets.py`)

**Fonts:** Download Big Shoulders Display 900 and Space Grotesk 500 TTFs from
Google Fonts CSS API at the top of the script using `urllib` (stdlib). Fall back
to DejaVu Sans Bold if the download fails (always offline-safe).

**Logo (pycairo):**
- 256×256 surface, transparent
- Draw `#0e0e0e` filled circle (r=120) centred
- Draw FoundryMark SVG paths (`M4 12 L28 12 ...`) scaled from 32×32 viewbox to
  ~160×160, centred on the disc, white stroke 5px, round caps/joins
- Export as PNG

**Banner (Pillow):**
- 800×200, `#0a0a0a` background
- Horizontal `#ff5b1a` rule 2px tall at y=140
- "FOUNDRY LINUX" in Big Shoulders Display 900 ~72px, `#f7f7f7`, left-anchored at x=40
- "26.04 LTS · ANVIL" in Space Grotesk 500 ~20px, `#ff5b1a`, below rule

**Slides (Pillow, all 800×440):**
- Background: crop centre of wallpaper resized to 800×440, darken by multiplying
  with 0.4 overlay, so text is legible but the forge glow is still visible
- `#ff5b1a` rule 2px at y=380
- Title text ~32px Space Grotesk Bold, `#ff5b1a`
- Body text ~16px Space Grotesk, `#ebe8e2`, word-wrapped to 700px wide
- Content per slide:
  1. "The Forge" / "WorldFoundry GDK, retro toolkit, Blender add-ons — struck in from first boot."
  2. "Stay current." / "apt.foundrylinux.org keeps every Foundry tool signed, rebuilt, and re-tested on every push."
  3. "Distrobox-ready." / "ghcr.io/foundry-linux/devbox:26.04 — the same environment, containerised."
  4. "Where to go next." / "foundrylinux.org/docs — package catalogue, install guide, contribution notes."

**GRUB background (Pillow):**
- Load wallpaper, resize to 1920×1080 (Lanczos, centre-crop to fill)
- Darken slightly (multiply 0.85) so GRUB text is readable
- Save as `data/config/grub/background.png`

**Plymouth copies:**
- `background.png` = same 1920×1080 as GRUB background (Pillow copy)
- `logo.png` = same 256×256 as branding logo (file copy)

**Plymouth .script (written directly):**
Simple fade-in of the logo centred on the background, with a spinner arc in `#ff5b1a`. Uses Plymouth's built-in Script module API.

**SDDM metadata.desktop (written directly):**
```ini
[SddmGreeterTheme]
Name=Foundry Linux
Description=Foundry Linux 26.04 login screen
Author=Foundry Linux
Version=1.0
Website=https://foundrylinux.org
Screenshot=preview.png
MainScript=Main.qml
```

**SDDM Main.qml (written directly):**
Minimal but styled — full-screen wallpaper background, centred login form with `#0e0e0e` semi-transparent panel, `#ff5b1a` accent border, white text. Reuses the `components/` from SDDM's default theme for the actual form logic (avoids reimplementing password entry, user list, etc.).

## Execution order

1. Run `python3 scripts/generate-assets.py` from the package root
2. Edit `branding.desc` accent color
3. `rm` all `ASSETS.md` placeholders
4. Run `task build` in `foundry-apt/` to verify the .deb builds cleanly
5. Check `dpkg-deb --contents` shows all assets at correct install paths
