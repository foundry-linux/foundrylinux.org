# Drop `linux` from `foundry-linux-*` package and script names

## Context

The package namespace `foundry-linux-*` carries a redundant infix. The distro is called "Foundry Linux" — that's branding — but the *package units* don't need to repeat it: every package in `apt.foundrylinux.org` is already for Foundry Linux by definition. The infix lengthens every `apt install` line and every script name without adding information. Trim it: `foundry-linux-retro-tools` → `foundry-retro-tools`, etc.

What stays:
- **Distro display name** "Foundry Linux" — branding, never changes
- **GitHub org** `foundry-linux` — identifies the distro at the source-host level
- **Domain** `foundrylinux.org` and apt host `apt.foundrylinux.org` — public surface
- **`worldfoundry-*` packages** — untouched (this rename is foundry-only)

What changes:
- 5 existing metapackages: `foundry-linux-{retro-tools, android-development, ios-development}` plus any `foundry-linux-{dev, blender, engine-build-deps}` that have been added since
- 6 install scripts: `install-foundry-linux-X.sh` → `install-foundry-X.sh`
- Directory `foundry-linux-setup/` → `foundry-setup/`
- All references in `CLAUDE.md`, `TODO.md`, Taskfiles, GitHub Actions workflows, READMEs, in-flight plans

## Prerequisites

- **apt.worldfoundry.org builder migration to ubuntu:26.04** ([../../worldfoundry.org/docs/plans/2026-05-21-apt-builder-ubuntu-26-04.md]) must publish a clean tag first. Zero file overlap with this rename, but bundling them risks a confused changelog history per package. Land that, verify, then start this.
- No file overlap with the `/packages` plan or the Phase 2 devbox plan, but those plans should retcon to the new names *after* this rename ships. Sequencing: rename → `/packages` → devbox image.

## Approach

### Mechanics — hard rename, one source dir per package

Each renamed source package keeps its single-binary structure; the old name disappears entirely. apt sees the old `foundry-linux-X` simply leave the index — anyone who installed it ends up with a dangling local package that `apt purge foundry-linux-X` removes cleanly, and they re-install under the new name.

Example — `foundry-apt/packages/foundry-retro-tools/debian/control`:

```
Source: foundry-retro-tools
Section: metapackages
Priority: optional
Maintainer: Foundry Linux <packages@foundrylinux.org>
Build-Depends: debhelper-compat (= 13)
Standards-Version: 4.6.2

Package: foundry-retro-tools
Architecture: all
Depends: mame, mame-tools, dasm, cc65, z80dasm, z80asm, radare2, binwalk,
         sox, binutils-m68k-linux-gnu, xa65, f9dasm, libvgm, vgmstream, ghidra
Description: Foundry Linux retro-tools metapackage.
 Pulls the full Foundry Linux retro/RE toolchain — disassemblers, emulators,
 audio tools, and Ghidra.
```

### Changelog continuity

Keep each package's existing changelog history. Add one new top entry per package:

```
foundry-retro-tools (1.0.5) resolute; urgency=medium

  * Rename from foundry-linux-retro-tools.

 -- Will Norris <will@foundrylinux.org>  Thu, 21 May 2026 14:30:00 +0000
```

Version bumps in the existing series so apt history is continuous across the rename. `dch -v <next> -D resolute "Rename from foundry-linux-X."` per package.

### Execution — one committed script

`scripts/rename-drop-linux.sh` does everything in one pass:

```bash
#!/usr/bin/env bash
set -euo pipefail

# Package source dirs
for old in foundry-apt/packages/foundry-linux-*; do
  new="${old/foundry-linux-/foundry-}"
  git mv "$old" "$new"
  # Rewrite Source: and Package: lines in debian/control
  sed -i 's/^\(Source\|Package\): foundry-linux-/\1: foundry-/' "$new/debian/control"
  # Bump changelog
  ( cd "$new" && dch -v "$(dpkg-parsechangelog -SVersion | next-patch)" \
      -D resolute "Rename from ${old##*/}." )
done

# Install scripts
for s in foundry-linux-setup/install-foundry-linux-*.sh; do
  git mv "$s" "${s/install-foundry-linux-/install-foundry-}"
done

# Setup directory
git mv foundry-linux-setup foundry-setup

# Doc/config sweep — everything else
git ls-files | grep -Ev '^(foundry-apt/packages/.*/debian/changelog|scripts/rename-drop-linux.sh)$' \
  | xargs sed -i 's/foundry-linux-/foundry-/g; s|foundry-linux-setup|foundry-setup|g'
```

**Run once, eyeball `git diff`, commit atomically.** No phased rollout — every old name disappears in the same commit.

## Verification

1. **Diff sanity**
   ```bash
   git diff --stat | tail -5
   git diff --name-only | grep -c foundry-linux-
   ```
   PASS = stat shows ~60 files changed; `grep -c foundry-linux-` returns 0 (every old name is gone from tracked content except the new transitional `Package:` stanzas — and those are *additions*, not pre-existing references).

2. **Every package builds**
   ```bash
   docker run --rm -v "$PWD:/work" -w /work ubuntu:26.04 bash -c 'bash foundry-apt/scripts/build-all.sh'
   ls -1 foundry-apt/dist/*.deb | wc -l
   ```
   PASS = same count as before the rename; every source dir produced its single renamed .deb.

3. **Renamed package structure**
   ```bash
   dpkg-deb -I foundry-apt/dist/foundry-retro-tools_*.deb | grep -E '^(Package|Depends|Description)'
   ```
   PASS = `Package: foundry-retro-tools`, same Depends list the old name had, same Description.

4. **Old name is gone from the index**
   ```bash
   # After tag + CI publish:
   docker run --rm -it ubuntu:26.04 bash -c '
     apt-get update && apt-get install -y curl gnupg ca-certificates
     curl -fsSL https://apt.foundrylinux.org/key.gpg \
       | gpg --dearmor -o /etc/apt/keyrings/foundry.gpg
     echo "deb [signed-by=/etc/apt/keyrings/foundry.gpg] https://apt.foundrylinux.org resolute main" \
       > /etc/apt/sources.list.d/foundry.list
     apt-get update
     apt-get install -y foundry-retro-tools           # new name works
     apt-cache show foundry-linux-retro-tools || true # old name returns E: Unable to locate
   '
   ```
   PASS = `apt-get install foundry-retro-tools` succeeds; `apt-cache show foundry-linux-retro-tools` reports "Unable to locate".

5. **Setup script smoke test**
   ```bash
   bash foundry-setup/install.sh --dry-run
   bash foundry-setup/install-foundry-retro-tools.sh --dry-run
   ```
   PASS = no missing-file errors; install plan references the new package names only.
