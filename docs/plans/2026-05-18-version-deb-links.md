# Plan: Link version numbers to .deb files in apt index

## Context

`apt.foundrylinux.org` shows a packages table generated by `generate-index.sh`. Version numbers are currently plain text. Making them links to the actual `.deb` files lets users inspect or manually download packages without knowing the APT pool URL structure.

## Decision: Yes, link them

Aptly publishes packages into a deterministic pool path:

```
/pool/main/<first-letter-of-name>/<name>/<name>_<version>_<arch>.deb
```

All metapackages (`foundry-linux-retro-tools`, `worldfoundry-*`) use `Architecture: all`, so their URLs are stable and unambiguous. The vendored `task` package has arch-specific debs (amd64, arm64) and needs special handling.

## Approach

### Critical file

`foundry-apt/scripts/generate-index.sh` — only file to change.

### 1. Parse Architecture from each control file

Add a `PKG_ARCHS` array alongside the existing parallel arrays (after line 29):

```bash
arch=$(grep '^Architecture:' "$control" | sed 's/^Architecture: *//' || true)
PKG_ARCHS+=("${arch:-all}")
```

### 2. Build version cell with pool link

Replace `<td>${ver}</td>` (line 45) with a version cell built from the pool URL:

- For `Architecture: all`:
  ```
  /pool/main/<L>/<name>/<name>_<ver>_all.deb
  ```
  → `<a href="…">1.0.0</a>`

- For arch-specific (e.g. `task`):
  ```
  /pool/main/<L>/<name>/<name>_<ver>_amd64.deb
  /pool/main/<L>/<name>/<name>_<ver>_arm64.deb
  ```
  → `1.0.0 (<a href="…">amd64</a> <a href="…">arm64</a>)`

### 3. No CSS changes needed

Blue link colour already applies via `a { color: var(--blue); }`. The version column's `--dim` styling stays; the `<a>` inside it picks up blue naturally.

## Verification

1. `task build && task publish-local` — builds debs, publishes to `./public/`
2. `python3 -m http.server 8080 --directory public` — serve locally
3. Open `http://localhost:8080` — version numbers should appear as blue links
4. Click a version link for an `arch: all` package — browser should download the `.deb`
5. Confirm `task` row shows `amd64` and `arm64` sub-links
6. `task shellcheck` — no lint errors
