# Plan: Retire the foundry-apt xa65 package

**Date:** 2026-05-18
**Status:** Complete (2026-05-19)

## Context

[`2026-05-18-package-xa65.md`](2026-05-18-package-xa65.md) packaged xa65 as `foundry-apt`'s first non-meta `.deb`. Its postscript records the lesson: **xa65 is already in Ubuntu 26.04 universe** as `2.4.1-0.1build1`. Our package (`2.4.1-1foundry1`) duplicated upstream effort and only persists by winning the version sort.

xa65 has served as the working reference implementation that drove every subsequent decision in this repo:
- Surfaced the static-`DEBIAN/control` problem in `generate-index.sh` (since fixed by the canonical-debian-layout migration).
- Drove the `Architecture: any` vs concrete-arch issue in the index generator (fixed in commit `139a674`).
- Motivated the `/package` skill (debhelper-based instead of hand-rolled `dpkg-deb --build`).

With those design problems solved and `f9dasm` now serving as the canonical reference for `/package`-built packages, **xa65's role is done**. Retire it and let Phase 0 + the metapackage `Depends:` chain pull the Ubuntu-universe xa65 directly.

## What changes

### 1. Delete `foundry-apt/packages/xa65/` entirely

Two files: `DEBIAN/control` and `build.sh`. Removing the directory has these consequences:

- `build-all.sh` no longer builds xa65 → `dist/xa65_*_amd64.deb` no longer appears in CI artifacts.
- aptly's next publish re-picks-up `dist/` and drops xa65 from the published Packages file → `apt-cache policy xa65` on a fresh container will show only Ubuntu's `2.4.1-0.1build1` candidate (not our `2.4.1-1foundry1`).
- The .deb still lives in the R2 pool at `pool/main/x/xa65/xa65_2.4.1-1foundry1_amd64.deb` (R2 sync is additive on most setups) but it's orphaned — not referenced by any Packages manifest. Acceptable: dead bytes, not security-relevant. We can scrub later if needed.

### 2. Add xa65 to Phase 0 apt-install list

`foundry-linux-setup/install-foundry-linux-retro-tools.sh` currently:
- Has a comment at line 98 saying *"xa65 is not in current Ubuntu repos under that name"* — wrong; remove.
- Has a source-build block at lines 137-143 cloning `github.com/fachat/xa65` and `make`-ing into `~/opt/xa65/` — drop.
- Has help text mentioning *"source-built xa65"* — update.

Replace by adding `xa65` to the `apt-get install -y` list (alongside `mame`, `dasm`, `cc65`, etc.). Phase 0 already enables `universe` implicitly (`xa65` is in universe; the script's existing apt operations resolve against `universe`).

### 3. Metapackage `Depends:` stays put

`packages/foundry-linux-retro-tools/debian/control` has `xa65` in `Depends:` (added in version 1.0.1). **Keep it.** apt will resolve it from Ubuntu universe once foundry-apt's xa65 is gone. No `Depends:` change, no metapackage version bump.

  - Drop the `xa65 (legacy)` row from the vendored-upstream table.
  - The `foundry-linux-retro-tools` row's prose mentions `xa65` — keep (it *does* still install xa65, just from universe).

### 5. `generate-index.sh` — no change needed

The DEBIAN/`control` fallback was already removed in commit `f1ddbb0` as part of the canonical-debian-layout migration. xa65 was the last consumer of that fallback; with xa65 gone, the simplification is complete.

### 6. `build-all.sh` — no change needed

It still supports the `build.sh` wrapper path (used by f9dasm and future vendored upstreams). xa65 was using `build.sh` too, but its directory is going away — the dispatch logic doesn't need a code change.

## Verification

1. **`packages/xa65/` is gone.**

    ```bash
    [[ -d foundry-apt/packages/xa65 ]] && echo FAIL || echo OK
    ```

2. **`task build` produces 6 .debs (5 metapackages + f9dasm), not 7.**

    ```bash
    ( cd foundry-apt && task build ) | grep -c "^OK"
    # expect: 6
    ls foundry-apt/dist/xa65*.deb 2>&1
    # expect: "No such file or directory"
    ```

3. **`generate-index.sh` produces 6 rows.**

    ```bash
    PUBLISH_DIR=/tmp/idx-test bash foundry-apt/scripts/generate-index.sh
    grep -cE "<tr><td>[^<]" /tmp/idx-test/index.html
    # expect: 7 (1 thead + 6 packages: 5 meta + f9dasm; no xa65)
    ```

4. **Phase 0 retro-tools dry-run mentions `xa65` in the apt-get install line, no source-build for xa65.**

    ```bash
    bash foundry-linux-setup/install-foundry-linux-retro-tools.sh --dry-run 2>&1 \
      | grep -E "apt-get install.*xa65|xa65.*git clone"
    # expect: one line, the apt-get install — no git clone
    ```

5. **Live (after sync + bump): `apt install xa65` resolves from Ubuntu universe.**

    ```bash
    docker run --rm ubuntu:26.04 bash -c '
        apt-get update -qq
        # add foundry-apt source (as the foundry-apt README documents)
        apt-get install -y -qq curl gnupg ca-certificates >/dev/null 2>&1
        install -d /etc/apt/keyrings
        curl -fsSL https://apt.foundrylinux.org/key.gpg \
          | gpg --dearmor -o /etc/apt/keyrings/foundry.gpg
        echo "deb [signed-by=/etc/apt/keyrings/foundry.gpg] https://apt.foundrylinux.org resolute main" \
          > /etc/apt/sources.list.d/foundry.list
        apt-get update -qq
        apt-cache policy xa65
    '
    # expect: only the Ubuntu universe candidate (2.4.1-0.1build1), no foundry-apt candidate
    ```

6. **`apt install foundry-linux-retro-tools` (metapackage) still installs xa65 transitively from Ubuntu universe.**
