# Plan: Link version numbers to .deb files in apt index

## Context

`apt.foundrylinux.org` shows a packages table generated by `generate-index.sh`. Version numbers are currently plain text. Making them links to the actual `.deb` files lets users inspect or manually download packages without knowing the APT pool URL structure.

## Decision: Yes, link them

Aptly publishes packages into a deterministic pool path:

```
/pool/main/<first-letter-of-name>/<name>/<name>_<version>_<arch>.deb
```

All metapackages (`foundry-retro-tools`, `worldfoundry-*`) use `Architecture: all`, so their URLs are stable and unambiguous. The vendored `task` package has arch-specific debs (amd64, arm64); those need special handling.

## Approach

### 1. Parse Architecture from control files

In `generate-index.sh`, add a `PKG_ARCHS` array alongside the existing parallel arrays (lines 14-30):

```bash
arch=$(grep '^Architecture:' "$control" | sed 's/^Architecture: *//' || true)
PKG_ARCHS+=("${arch:-all}")
```

### 2. Build version cell with link

Replace the plain `${ver}` in the table row (line 45) with a helper that constructs the pool URL:

```bash
# pool path: pool/main/<L>/<name>/<name>_<ver>_<arch>.deb
L="${name:0:1}"
if [[ "$arch" == "all" ]]; then
  deb_url="/pool/main/${L}/${name}/${name}_${ver}_all.deb"
  ver_cell="<a href=\"${deb_url}\">${ver}</a>"
else
  # arch-specific: show small download links per arch
  ver_cell="${ver} (<a href=\"/pool/main/${L}/${name}/${name}_${ver}_amd64.deb\">amd64</a> <a href=\"/pool/main/${L}/${name}/${name}_${ver}_arm64.deb\">arm64</a>)"
fi
```

The existing CSS already styles `td:nth-child(2)` as `color: var(--dim); white-space: nowrap` — the version + small arch links will fit cleanly.

### 3. No CSS changes needed

The blue link colour (`var(--blue)`) already applies via `a { color: var(--blue); }`. Version column stays `--dim` for the surrounding text; only the `<a>` tag inside goes blue.

## Critical file

- `foundry-apt/scripts/generate-index.sh` — only file to change

## Verification

1. Run `task build && task publish-local`
2. Open `public/index.html` in a browser — version numbers should be blue links
3. Click a version link — should 404 gracefully (local `./public/` has the file at the exact pool path, so serve `./public/` with `python3 -m http.server` and confirm the link resolves to a real `.deb`)
4. Confirm arch-specific packages show `amd64 | arm64` sub-links
5. Run `task shellcheck` — no lint errors
